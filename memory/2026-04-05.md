# Just a Child - Project Context (Updated)

## Tech Stack
Unity 2D (6000.x), URP, Cinemachine 2.10.7, DOTween, UniRx, Zenject DI, TMPro

## All Scripts & Their Purpose

### Core Game Flow
- `Assets/Scripts/GameFlowController.cs` — Orchestrates: intro (Cinemachine vcam priority swap + IntroTextView typewriter), 5min timer, win/lose conditions, scene restart after delay. Uses Awake() to set introVCam priority=20 immediately. Volume on playerVCam for vignette.
- `Assets/Scripts/Installers/GameInstaller.cs` — Zenject MonoInstaller. Binds: IPlayerView, PlayerModel, PlayerConfig, PlayerFearController, InputController, GameFlowController, FearAttractionManager, InspectionFeedbackView.

### Player
- `Assets/Scripts/PlayerModel.cs` — ReactiveProperty<float> CurrentFearLevel, IsFearMaxed, IncreaseFear/DecreaseFear/IncreaseFearBy/DecreaseFearBy
- `Assets/Scripts/Player/PlayerConfig.cs` — ScriptableObject: MaxFearValue, FearDecreaseValue, FearIncreaseValue, FearIncreaseInterval, FearDecreaseCooldown, InitialFearDelay
- `Assets/Scripts/PlayerFearController.cs` — Fear increase/decrease coroutines. InitialFearDelay before fear starts. Start/StopIncreasingFear, Start/StopDecreaseFearWithCooldown. Does NOT auto-start fear (GameFlowController calls StartIncreasingFear).
- `Assets/Scripts/Player/InputController.cs` — OnInteractionPerformed (Subject<Unit>) fires when interact pressed + InteractionAvailable
- `Assets/Scripts/Player/IPlayerView.cs` — Interface: ShowInteraction(bool), InteactionAvailable
- `Assets/Scripts/Player/PlayerView.cs` — Implements IPlayerView
- `Assets/Scripts/MovementController.cs` — WASD movement
- `Assets/Scripts/CharacterAnimatorController.cs` — Animation states
- `Assets/Scripts/CharacterSpriteFlipper.cs` — Flip sprite based on direction

### Furniture / Fear System
- `Assets/Scripts/Furniture/FearAttraction.cs` — Activates with random sound from AudioClip[] + DOTween shake. Passive fear increase via coroutine. Player investigates via trigger + InputController subscription to deactivate + reduce fear + PlaySuccess(). Has event OnInvestigated. SerializeField FearDecreaseArea _linkedDecreaseArea.
- `Assets/Scripts/Furniture/FearAttractionManager.cs` — Escalating difficulty: intervals decrease over time (_intervalDecreasePerSecond), simultaneous activations increase (_secondsPerExtraSlot, _maxSimultaneous=4). Continuous spawning loop. OnInvestigated triggers immediate TryActivateOne.
- `Assets/Scripts/Furniture/FearDecreaseArea.cs` — Has [SerializeField] FearAttraction _linkedAttraction. If linked attraction is active -> correct inspection (calls _linkedAttraction.Investigate()). Else -> wrong inspection (increases fear + PlayFail).
- `Assets/Scripts/Furniture/FearDecreasingLightSource.cs` — Available after _activationDelay (30s). Attract flicker when available. Player enters trigger -> stops flicker, enables light, stops fear increase, starts fear decrease. On exit -> restarts fear, disables light. After use: _reloadDelay then available again with attract flicker.

### UI
- `Assets/Scripts/UI/InspectionFeedbackView.cs` — URP Volume Vignette (on playerVCam) + CinemachineImpulseSource for shake + TMP_Text for "Wuhh". PlaySuccess() = green vignette + text. PlayFail() = red vignette + camera shake. Enables camera post-processing in Awake.
- `Assets/Scripts/UI/TimerView.cs` — MM:SS countdown display
- `Assets/Scripts/UI/IntroTextView.cs` — Typewriter text animation, inspector-configurable text, callback on complete
- `Assets/Scripts/FearMeterView.cs` — Fear bar UI

## Key Architecture Decisions
- Camera: Cinemachine 2.x, two VCams (intro + player). Priority swap for transitions. No manual parent/unparent.
- Vignette: URP Volume on playerVCam GameObject (Global mode), controlled via code. Camera post-processing enabled programmatically.
- Shake: CinemachineImpulseSource + ImpulseListener extension on playerVCam
- Fear system: PlayerFearController does NOT auto-start. GameFlowController calls StartIncreasingFear() after intro.
- FearAttraction: one-at-a-time initially, escalates to multiple over time
- Interaction: each component (FearDecreaseArea, FearAttraction) handles its own interaction via InputController subscription. PlayerFearController no longer auto-decreases on interact.
- Scene restart on win/lose via SceneManager.LoadScene after delay
- Sounds: FearAttraction picks random AudioClip from array on each Activate()
